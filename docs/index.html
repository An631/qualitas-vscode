Team Configuration

Analysis rules live in qualitas.config.js and are committed to the repo. VS Code settings control only extension behavior. Same pattern as ESLint.

  • Profiles: default, strict, cc-focused, data-focused
  • Per-language flag overrides
  • Custom weights for the 5 pillars
  • TypeScript, JavaScript, Python, Rust
// qualitas.config.js
module.exports = {
  profile: "strict",
  threshold: 80,
  flags: {
    tooManyParams: { warn: 3, error: 5 },
    excessiveReturns: false,
  },
  languages: {
    typescript: {
      flags: { deepNesting: { warn: 3, error: 5 } }
    }
  }
};

Getting Started

Use the VS Code extension for real-time feedback, or the npm package for CLI and CI

1

Install the extension

Install from the VS Code Marketplace or download the .vsix from GitHub Releases.

code --install-extension qualitas-vscode-x.x.x.vsix
2

Open a project

Open any TypeScript, JavaScript, Python, or Rust project. Qualitas activates automatically.

3

See scores instantly

Every function gets a CodeLens score. Functions with issues get squiggly underlines.

4

Fix with AI

Click the lightbulb on any squiggled function and choose "Fix with AI" to open your AI chat.

5

Configure

Add a qualitas.config.js to your project root. The whole team shares the same rules.

module.exports = { profile: "strict", threshold: 80 };
1

Install

Add qualitas as a dev dependency.

npm install --save-dev qualitas
2

Analyze

Run qualitas on your project for a full quality report.

npx qualitas ./ -f detail
3

Add to scripts

Add quality checks to package.json for CI.

"scripts": { "quality": "qualitas ./ -f detail" }
4

Use the API

Import qualitas for custom analysis or automation.

const { analyzeSource } = require("qualitas");
const report = analyzeSource(source, "file.ts");
5

Configure

Create a qualitas.config.js to customize everything.

module.exports = { profile: "strict", threshold: 80 };
